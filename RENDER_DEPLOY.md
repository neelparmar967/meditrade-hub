# Deploy MediTrade Hub To Render

## Required Accounts

- Render account
- MongoDB Atlas database
- GitHub repository containing this project

## 1. MongoDB Atlas

Create a MongoDB Atlas cluster and copy the connection string.

Example:

```text
mongodb+srv://USER:PASSWORD@cluster0.xxxxx.mongodb.net/meditrade_hub
```

Use a real password and keep it private.

## 2. Push Project To GitHub

Render deploys from GitHub most easily. Push this folder as a GitHub repository.

## 3. Deploy With Render Blueprint

In Render:

1. Click **New**
2. Click **Blueprint**
3. Connect the GitHub repository
4. Render should detect `render.yaml`
5. Add `MONGO_URI` when Render asks for it
6. Deploy

This creates:

- `meditrade-hub-api`
- `meditrade-hub-web`

## 4. Check Environment Variables

Backend service:

```env
NODE_ENV=production
JWT_SECRET=<generated by Render>
MONGO_URI=<your MongoDB Atlas URI>
CLIENT_ORIGIN=https://meditrade-hub-web.onrender.com
```

Frontend service:

```env
VITE_API_BASE_URL=https://meditrade-hub-api.onrender.com/api
```

If Render changes your service URLs, update:

- Backend `CLIENT_ORIGIN`
- Frontend `VITE_API_BASE_URL`

## 5. Seed Demo Data

Open the Render backend service shell and run:

```bash
npm run seed
```

Only run seed when you want demo accounts/products inserted.

## 6. Test

Open:

```text
https://meditrade-hub-web.onrender.com
```

Check backend health:

```text
https://meditrade-hub-api.onrender.com/api/health
```

Expected:

```json
{"ok":true,"database":"mongo"}
```

## Manual Render Settings

If you do not use Blueprint:

Backend Web Service:

```text
Root Directory: server
Build Command: npm install
Start Command: npm start
Health Check Path: /api/health
```

Frontend Static Site:

```text
Root Directory: client
Build Command: npm install && npm run build
Publish Directory: dist
```
